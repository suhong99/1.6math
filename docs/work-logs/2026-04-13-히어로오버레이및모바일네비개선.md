## 진행 상황

### 2026-04-13

#### Hero 오버레이 강화
- `bg-slate-900/80` → `bg-slate-900/95`로 변경
- Tailwind v4에서 `/92` 같은 임의 opacity 값은 적용 안 됨 주의 (정의된 값 또는 `[.92]` 형태 사용)

#### 히어로 타이틀 Galaxy Fold 대응
- 기존: `text-5xl lg:text-7xl`
- 변경: `text-3xl min-[380px]:text-5xl lg:text-7xl`
- Galaxy Fold 폴딩 시 (~280px) `text-3xl`, 일반 폰 `text-5xl`, 데스크탑 `text-7xl`

#### 모바일 사이드바 네비 가시성 개선
- 비활성 항목: `bg-slate-200 border border-slate-300` 배경+테두리 추가
- 각 항목에 아이콘 추가 (학원소개=집, 오시는길=핀, 익명건의함=말풍선)
- 아이콘 색: 비활성 `text-slate-400`, 활성 `text-white`

#### 사이드바 열릴 때 뒤 콘텐츠 선택 방지
- `useEffect`로 isOpen 시 `document.body`에 `overflow-hidden select-none` 토글

#### 가로 스크롤 시 닫힌 사이드바 노출 방지
- `layout.tsx` 루트 div에 `[overflow-x:clip]` 추가
- `overflow: hidden` 대신 `clip` 사용 → 스크롤 컨테이너 생성 없이 클립만 적용, `fixed` 요소에 영향 없음

#### 오버레이 전체 화면 미적용 버그 수정 (핵심)
- **원인**: MobileSidebar가 `<header z-[800]>` 안에 있어 오버레이의 z-[999]가 Header 스태킹 컨텍스트에 갇힘. 루트 기준 z-800으로만 작동해 헤더 아래 DOM 후반 콘텐츠가 오버레이 위에 렌더링됨
- **해결**: `createPortal(…, document.body)`로 오버레이와 드로어를 body에 직접 마운트 → 루트 스태킹 컨텍스트에서 z-999 정상 작동
- SSR 대응: `mounted` 상태로 `document.body` 접근 시점 보호

#### 헤더 로고 모바일 표시
- `hidden sm:block` (640px 이상만) → `hidden min-[320px]:block` (320px 이상)
- Galaxy Fold 폴딩(~280px) 제외 모든 모바일에서 "1.6수학과학전문학원" 전체 표시
